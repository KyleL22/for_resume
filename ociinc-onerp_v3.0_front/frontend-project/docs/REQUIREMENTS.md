# 경영지원팀 인적 프로필 및 프로젝트 통합 관리 시스템 요구사항 정의서

## 1. 프로젝트 개요
- **프로젝트명:** 경영지원팀 인적 프로필 및 프로젝트 통합 관리 시스템
- **목적:** 효율적인 인력 관리, 프로젝트 투입 이력 추적, 비정형 데이터(이력서 등)의 자동화된 메타데이터 확보 및 견적 관리 시스템 구축
- **주요 사용자:** 경영지원팀(관리자), 영업 대표, 일반 직원

## 2. 기술 스택 (Tech Stack)
- **Frontend:** React.js
- **Backend:** Python (초기 Jupyter 기반 테스트 및 개발 후 FastAPI/Flask 서버 이전)
- **Database:** Supabase (PostgreSQL 기반)
- **Infrastructure:** Azure OpenAI(문서 파싱 및 분석), Azure/GCP(백엔드 호스팅)

## 3. 상세 요구사항
### 3.1 직원 및 인력 관리 (HR Management)
- **권한 기반 인증 시스템:** 일반 사용자, 경영지원팀(관리자), 영업 대표로 구분된 로그인 기능.
  - 회원 가입은 보안을 위해 관리자가 직접 정보를 입력하여 계정을 부여하는 방식으로 제한.
- **인적 프로필 데이터화:**
  - 개인정보 암호화: 성명, 주민등록번호, 연락처, 계좌번호 등 민감 정보는 DB 내 암호화 저장.
  - 메타데이터 관리: 학력, 기술 등급(초/중/고/특급), 경력 년수, 자격증, 최종 학교 등 관리.
- **비정형 데이터 확보:**
  - 잡코리아 등 외부에서 확보한 이력서 파일(JPG, PDF, Word 등) 업로드 기능.
  - Azure OpenAI를 활용하여 파일 내 텍스트를 추출하고 시스템 메타데이터와 자동 매핑.

### 3.2 프로젝트 관리 (Project Management)
- **프로젝트 목록 관리:** 프로젝트명, 기간, 발주처, 업종, 계약금액, 계약처 구분(운영/개발) 관리.
- **투입 이력 관리:**
  - 인력별 프로젝트 참여 이력(수행 기간, 역할, 적용 기술, 평가 등) 자동 기록.
  - 기존 입력된 데이터를 최초 메타데이터로 사용하며, 신규 투입 시 실시간 매핑.
  - **매핑 방식 (Drop & Insert):** 투입 프로젝트 정보 업데이트 시 기존 정보를 바탕으로 새로운 매핑 데이터를 생성하고 관리.

### 3.3 견적 관리 (Estimation - 1단계 이후 추가 예정)
- **견적 시스템:** 프로젝트별 투입 인력의 기술 등급에 따른 매입/매출 단가 산출.
- **우선순위:** 1단계에서는 프론트엔드 UI/UX 중심 선제작.

### 3.4 데이터 아키텍처 및 매핑
- **기본 정보 연동:** `member` 테이블과 `profile` 테이블의 1:1 매핑을 통한 상세 정보 관리.
- **코드 표준화:** 업종 코드, 직위 코드, 기술 등급 코드 등 공통 코드 테이블(`code_group`, `code_detail`)을 참조하여 데이터 무결성 유지.

## 4. 데이터베이스 설계 (ERD 기준)
- **Member:** 사용자 계정 및 기본 신상 정보.
- **Profile:** 전문 기술 정보 및 원문 파일 내용.
- **Project:** 프로젝트 마스터 정보.
- **Project_User:** 프로젝트별 인력 투입 현황.
- **Project_Perform:** 개인별 프로젝트 수행 실적 이력.
- **Estimate / Estimate_Detail:** 견적서 및 상세 항목 관리.

## 5. 보안 및 규정 준수
- **개인정보 보호:** 주민등록번호 등 식별 정보는 반드시 암호화 처리하고 화면 노출 시 마스킹 처리.
- **접근 제어:** 관리자(경영지원팀) 페이지는 별도 권한을 가진 사용자만 접근 가능하도록 제한.
